## Overview

The Credit Scoring Service is a RESTful microservice that predicts the credit risk of a loan applicant. It performs **binary classification** — outputting either `good` or `bad` credit risk — using the [German Credit Risk dataset](https://www.kaggle.com/datasets/uciml/german-credit) as its training basis.

The service receives demographic and financial data, processes it through a scikit-learn preprocessing pipeline, and feeds it to a trained MLP model. It returns a prediction label and the associated probability score.

## Model

The service uses a **Multi-Layer Perceptron (MLP)** built with PyTorch, optimized with:

- **Hidden layers:** `[256, 128, 64, 64]`
- **Dropout rate:** `0.1`
- **Batch Normalization:** enabled
- **Activation function:** ReLU
- **Input features:** 26 (after preprocessing/encoding)

```python
BEST_MODEL_CONFIG = {
    'num_features': 26,
    'hidden_layers': [256, 128, 64, 64],
    'dropout_rate': 0.1,
    'use_batch_norm': True,
    'activation_fn': 'ReLU'
}
```

The model is loaded once at startup (singleton pattern) and runs in `eval()` mode for inference.

## Endpoint

### `POST /mlp_demo`

Receives applicant data and returns a credit risk prediction.

**Request body** — `CreditRiskInput`:

```json
{
  "Age": 35,
  "Sex": "male",
  "Job": 2,
  "Housing": "own",
  "Saving accounts": "little",
  "Checking account": "moderate",
  "Credit amount": 2500,
  "Duration": 24,
  "Purpose": "car"
}
```

**Response** — `CreditRiskOutput`:

```json
{
  "prediction": "good",
  "probability": 0.7852
}
```

| Field | Type | Description |
|---|---|---|
| `prediction` | string | `"good"` or `"bad"` |
| `probability` | float `[0, 1]` | Probability that the risk is `"good"` |

<Note>
  The threshold for classification is `0.5`. If `probability >= 0.5`, the prediction is `"good"`.
</Note>

### `GET /`

Redirects to the interactive Swagger UI at `/docs`.

## Local Development

<Steps>
  <Step title="Prepare model artifacts">
    Place the trained model (`.pt`) and preprocessor (`.joblib`) in:
    ```
    python/credit_scoring/models/
    ```
  </Step>
  <Step title="Build the Docker image">
    From the repository root (`deep_learning_services/`):

    ```bash
    docker build -t ingeniia/credit-scoring-mlp:1.0 -f container-images/credit_scoring_service/Dockerfile .
    ```
  </Step>
  <Step title="Run the container">
    ```bash
    docker run -d -p 8000:8080 --name credit-scoring-service ingeniia/credit-scoring-mlp:1.0
    ```
  </Step>
  <Step title="Verify the service">
    Open the interactive API documentation in your browser:

    ```
    http://localhost:8000/docs
    ```
  </Step>
</Steps>

## Container Management

```bash
# Stop the container
docker stop credit-scoring-service

# Stream logs in real time
docker logs -f credit-scoring-service

# Restart a stopped container
docker start credit-scoring-service

# Stop and remove the container
docker stop credit-scoring-service && docker rm credit-scoring-service
```
